# Enrolment Funnel Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Redesign the parent-facing enrolment funnel at power2adapt.online -- quiz-led class discovery, streamlined 2-step enrolment form, waitlist with holiday program bonus, sibling discount, and post-payment confirmation page.

**Architecture:** All schema changes are additive (new columns only, no existing columns modified). New React components sit alongside existing ones. Backend changes are new routes and new logic in existing routes -- existing enrolment flow is not broken. Plan 2 covers admin features, re-enrolment window, makeup classes, and PWA attendance.

**Tech Stack:** React 18 + TypeScript, Wouter routing, TanStack Query, shadcn/ui, Tailwind CSS, Node.js + Express, Drizzle ORM, PostgreSQL (DATABASE_URL env var), Stripe, Twilio SMS, Resend email.

**Codebase entry points:**
- `shared/schema.ts` -- all Drizzle table definitions and enums
- `server/routes.ts` -- all Express API routes
- `server/storage.ts` -- database query functions
- `client/src/pages/` -- React page components
- `client/src/components/` -- shared React components

- [ ] **Step 1: Add `getClassesWithSpots` to `server/storage.ts`**

Find the existing function that fetches classes (likely `getClasses` or similar). Add a new function below it:

```typescript
export async function getClassesWithSpots() {
  const classRows = await db.select().from(classes);

  return classRows.map((cls) => ({
    ...cls,
    spotsRemaining: cls.maxCapacity - (cls.currentEnrollment ?? 0),
    spotsTaken: cls.currentEnrollment ?? 0,
  }));
}
```

**Note:** `classes.currentEnrollment` already exists in the schema as an integer field (default 0). Use it directly — no need to count enrollments rows.

- [ ] **Step 2: Add `getActiveEnrolmentCountForParent` to `server/storage.ts`**

This checks if a parent qualifies for the sibling discount (2+ active enrolments this term):

```typescript
export async function getActiveEnrolmentCountForParent(parentId: string, term: string, year: number): Promise<number> {
  const rows = await db
    .select()
    .from(enrollments)
    .innerJoin(classes, eq(enrollments.classId, classes.id))
    .where(
      and(
        eq(enrollments.parentId, parentId),
        eq(enrollments.status, 'active'),
        eq(classes.term, term as any),
        eq(classes.year, year)
      )
    );
  return rows.length;
}
```

**Note:** `parentId` is a `uuid` string — do NOT use `parseInt`.

- [ ] **Step 3: Update the GET `/api/classes` route in `server/routes.ts`**

Find the existing `GET /api/classes` handler. Replace the storage call with `getClassesWithSpots()`:

```typescript
router.get('/api/classes', async (req, res) => {
  try {
```

- [ ] **Step 4: Add `GET /api/classes/sibling-discount` route**

Add after the classes route:

```typescript
router.get('/api/classes/sibling-discount', async (req, res) => {
  if (!req.user) return res.json({ eligible: false, count: 0 });
  const { term, year } = req.query as { term: string; year: string };
  if (!term || !year) return res.json({ eligible: false, count: 0 });

  try {
    // req.user.id is a uuid string — pass directly, no parseInt
    const count = await getActiveEnrolmentCountForParent(
      req.user.id,
      term,
      parseInt(year, 10)   // year is safe to parse as integer
    );
    res.json({ eligible: count >= 2, count });
  } catch (error) {
    res.status(500).json({ error: 'Failed to check sibling discount' });
  }
});
```

- [ ] **Step 1: Add `createWaitlistWithHolidayReservation` to `server/storage.ts`**

**Critical:** The schema uses a separate `waitlists` table (not an enrollment status). All IDs are uuid strings.

The `waitlists` table columns: `id` (uuid pk), `classId`, `childId`, `parentId`, `position` (integer), `status` (waitlistStatusEnum: active/notified/enrolled/cancelled/expired), `joinedAt`, `notifiedAt`, `expiresAt`.

```typescript
export async function createWaitlistWithHolidayReservation(
  childId: string,
  classId: string,
  parentId: string
): Promise<{ waitlistEntry: any; holidayReservation: any | null }> {
  // Get current waitlist length to determine position
  const existingEntries = await db
    .select()
    .from(waitlists)
    .where(
      and(
        eq(waitlists.classId, classId),
        eq(waitlists.status, 'active')
      )
    );
  const nextPosition = existingEntries.length + 1;

  // Insert into waitlists table (not enrollments)
  const [waitlistEntry] = await db.insert(waitlists).values({
    classId,
    childId,
    parentId,
    position: nextPosition,
    status: 'active',
    // expiresAt is set when a spot opens and we send the SMS notification
  }).returning();

  // Find a holiday program matching the child's age
  const childRows = await db.select().from(children).where(eq(children.id, childId)).limit(1);
  if (!childRows[0]) return { waitlistEntry, holidayReservation: null };

  const childAge = new Date().getFullYear() - new Date(childRows[0].dateOfBirth).getFullYear();

  const holidayClass = await db
    .select()
    .from(classes)
    .where(
      and(
        eq(classes.isHolidayProgram, true),
        lte(classes.minAge, childAge),
        gte(classes.maxAge, childAge),
        eq(classes.isEnrollmentOpen, true)
      )
    )
    .limit(1);

  if (!holidayClass[0]) return { waitlistEntry, holidayReservation: null };

  // Reserve holiday program spot as a pending_payment enrollment
  const [holidayReservation] = await db.insert(enrollments).values({
    childId,
    classId: holidayClass[0].id,
    parentId,
    status: 'pending_payment',
    waitlistHolidayReservation: true,
  }).returning();

  return { waitlistEntry, holidayReservation };
}
```

## Task 10: Application-only gate for selective programs

**Added:** 23 May 2026

**Context:** Junior Academy, Senior Squad, and High Performance programs are selective — parents should not be able to self-enrol. When a parent taps "Enrol now" on one of these classes, they must be directed to the application form instead.

**Application-gated sport types:**
```typescript
const APPLICATION_ONLY_PROGRAMS = [
  'academy_year7_above',   // Junior Academy
  'senior_squad',          // Senior Squad
  'empowered_athlete_program', // High Performance / Empowered Athlete
  'competition_ready',     // Competition Ready
];
```

**Files:**
- Modify: `client/src/components/classes/class-card.tsx`
- Modify: `client/src/pages/enrollment.tsx`

**Behaviour — class-card.tsx:**
- Add a helper: `const isApplicationOnly = (sportType: string) => APPLICATION_ONLY_PROGRAMS.includes(sportType)`
- When `isApplicationOnly(classData.sportType)` is true, render an "Apply now" button instead of "Enrol now"
- Button style: same size/shape as "Enrol now" but use `bg-purple-600 hover:bg-purple-700` to visually distinguish it
- Button label: `Apply now →`
- On click: navigate to `/questionnaire?classId=${classData.id}&className=${encodeURIComponent(classData.name)}`
- This applies regardless of spots remaining — application-only classes never show a self-enrol path

**Behaviour — enrollment.tsx (guard):**
- At the top of the component, after `cls` is loaded, add a guard:
```typescript
const isApplicationOnly = APPLICATION_ONLY_PROGRAMS.includes(cls?.sportType);
```
- If `isApplicationOnly` is true, render a redirect screen instead of the enrolment flow:
  - Heading: "This is an application-only program"
  - Body: "Junior Academy, Senior Squad, and High Performance programs require an application. Our coaching team reviews all applications before offering a place."
  - Button: `Complete your application →` — links to `/questionnaire?classId=${classId}&className=${encodeURIComponent(cls.name)}`
  - Secondary link: `← Back to classes` — links to `/classes`

**Behaviour — questionnaire.tsx:**
- Read `classId` and `className` from the URL query string if present
- Pre-fill or display "Applying for: [className]" at the top of the form so the parent knows which program they're applying for
- On submission, include `classId` in the payload if present (the admin applications view can use this to link the application to the specific class)

**Note for implementer:**
- The `APPLICATION_ONLY_PROGRAMS` array should be defined in a shared constants file (e.g. `client/src/lib/constants.ts`) so class-card.tsx and enrollment.tsx both import from the same source — not duplicated
- The questionnaire already posts to the server; make sure `classId` and `className` are added to the submission payload and stored in the `applications` table (check `server/routes.ts` for the existing questionnaire POST handler)
